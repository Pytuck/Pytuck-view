# pytuck-view 开发计划

## 1. 表管理功能

### 1.1 表操作
- [ ] 删除表
- [ ] 清空表数据
- [ ] 新建表

### 1.2 表结构完整编辑（新建表的前置条件）

目前仅支持修改列备注，需扩展为完整的表结构编辑：

- [ ] 修改列类型（需支持自动类型转换）
  - int → str：自动转换
  - str → int：可转换则转换，不可转换（如 "abc"）则报错
  - 可能需要 pytuck 底层支持
- [ ] 修改主键设置
- [ ] 修改必填性（nullable）
- [ ] 修改默认值
- [ ] 添加新列
- [ ] 删除列
- [ ] 修改列顺序（可选）

**约束**：修改后若现有数据不满足新约束，应报错并拒绝变更。

---

## 2. 表搜索功能

- [ ] 左侧表列表支持搜索（前端过滤）
  - 按表名搜索
  - 按表备注搜索

---

## 3. 数据表视图优化

- [ ] 支持横向滚动查看多列数据
  - 当前列数过多时表格无法完整显示
  - 需确保表头与数据列对齐

---

## 4. 数据导入导出

- [ ] 导出数据
  - 导出整表数据
  - 导出格式：CSV、JSON、Excel 等
- [ ] 导入数据
  - 从文件导入到现有表
  - 格式匹配与数据验证

---

## 5. 数据库文件转换

- [ ] 在首页支持数据库文件格式转换
  - 在 pytuck 支持的引擎类型间互转
  - 如：binary ↔ json ↔ sqlite ↔ csv 等
  - 需要 pytuck 提供转换接口支持

---

## 6. 列筛选功能

- [ ] 在数据表格视图的表头上直接筛选
  - 输入搜索内容
  - 选择搜索方式（等于、包含、大于、小于等）
  - 支持多列组合筛选

---

## 优先级建议

### 近期（P0-P1）
1. **P0**：横向滚动（体验问题，影响基本使用）
2. **P1**：表搜索（易实现，提升效率）

### 中期（P2-P3）
3. **P2**：表结构完整编辑（工作量大，需 pytuck 配合）
4. **P3**：表管理功能（依赖表结构编辑完成）
5. **P3**：列筛选功能（提升数据查看效率）

### 远期（P4）
6. **P4**：数据导入导出
7. **P4**：数据库文件转换（需 pytuck 支持）
